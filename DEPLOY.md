# Как выложить Voyanté в интернет и делиться ссылкой

Сейчас сайт открывается только у вас: `http://localhost:3000`.  
Чтобы **любой человек** открыл сайт по ссылке (например `https://voyante.vercel.app`), нужно **опубликовать** проект на хостинге.

---

## Способ 1 — Vercel (рекомендуется для Next.js)

Бесплатно, подходит для этого проекта (страницы + API).

### Шаг 1. Установите Git (один раз)

1. Скачайте: https://git-scm.com/download/win  
2. Установите с настройками по умолчанию.  
3. Перезапустите Cursor / терминал.

### Шаг 2. Создайте репозиторий на GitHub

1. Зайдите на https://github.com и войдите в аккаунт.  
2. **New repository** → имя, например `voyante-travel` → **Create**.  
3. В терминале в папке проекта:

```powershell
cd "c:\Users\Admin\cursor 1"
git init
git add .
git commit -m "Voyante travel site"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/voyante-travel.git
git push -u origin main
```

(подставьте свой логин и URL репозитория)

### Шаг 3. Подключите Vercel

1. https://vercel.com → войти через GitHub.  
2. **Add New Project** → выберите репозиторий `voyante-travel`.  
3. Framework: **Next.js** (определится сам).  
4. **Deploy**.

Через 1–2 минуты получите ссылку вида:

`https://voyante-travel.vercel.app`

Её можно отправлять в WhatsApp, Telegram, email.

### Обновление сайта после правок

```powershell
git add .
git commit -m "Update"
git push
```

Vercel пересоберёт сайт автоматически.

## Способ 2 — Без Git (через сайт Vercel + ZIP)

Если Git пока не хотите ставить:

1. Заархивируйте папку `c:\Users\Admin\cursor 1` в ZIP.  
2. **Исключите** из архива: `node_modules`, `.next` (они большие и не нужны).  
3. На https://vercel.com можно импортировать проект через GitHub — без Git проще всё же установить Git (способ 1).

## Что уже «внутри» проекта (не нужно отдельно загружать)

| Файлы | Где лежат | На опубликованном сайте |
|--------|-----------|-------------------------|
| Фото на главной | `public/images/` | Откроются сами |
| YouTube-реклама | в коде (`lib/promo-videos.ts`) | Работает по интернету |
| Генератор / чат | API на сервере Vercel | Работает без ключей (mock) |

Видео `video.mp4` в корне — **не обязательно**; hero использует градиент, если файла нет.

## Временная ссылка без публикации (только для теста)

Пока сайт только на вашем ПК, в той же Wi‑Fi сети можно открыть с телефона:

В терминале при `npm run dev` смотрите строку **Network:**, например:

`http://172.20.10.3:3000`

Это **не** публичная ссылка в интернете — только в вашей сети.
